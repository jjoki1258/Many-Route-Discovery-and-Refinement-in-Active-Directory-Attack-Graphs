# Many-Route-Discovery-and-Refinement-in-Active-Directory-Attack-Graphs

 1. Overall Summary of the Paper

<img width="953" height="435" alt="image" src="https://github.com/user-attachments/assets/86ac1239-548b-4aa3-b34c-63636bcb2686" />

This paper proposes a novel attack path exploration framework for analyzing multistep attacks in Active Directory (AD)–based enterprise environments. Existing attack graph studies have primarily focused on identifying a single optimal attack path or the shortest attack path. However, real attackers rarely rely on only one route. Instead, they continuously explore and maintain multiple alternative attack paths that can be used depending on the network state or defensive responses. The paper argues that traditional approaches fail to sufficiently capture this realistic attacker behavior.

To address this limitation, the paper introduces a framework that combines a Global Route-Bank Genetic Algorithm with a Two-Stage Fast Local Search mechanism. The framework consists of three major stages. First, the Active Directory environment is modeled as an attack graph in which users, groups, computers, organizational units (OUs), and group policy objects (GPOs) are represented as nodes, while privilege delegation and lateral movement relationships are represented as directed edges. Second, a global exploration process based on a Genetic Algorithm is performed. In this stage, diverse attack paths are generated using random walks and BFS-based initialization strategies. Genetic operators such as crossover and mutation continuously produce new candidate attack paths. In particular, the proposed route-bank structure preserves diverse attack routes by filtering structurally similar paths and maintaining only sufficiently different ones.

The third stage applies a Two-Stage Local Search refinement process to improve the quality of the discovered attack paths. Stage 1 performs lightweight refinement operations such as cycle cutting, cutoff-based shortening, and local replacement to quickly simplify attack paths. Stage 2 then applies weighted shortest-path reconstruction to optimize the paths more precisely, aiming to increase attack success probability while reducing attack cost and detectability.

A key contribution of the paper is that attack path quality is evaluated using a multi-objective optimization perspective rather than simple shortest-path metrics. The framework simultaneously considers attack success probability, attack cost, detectability, and path length through a unified fitness function. In addition, Jaccard similarity–based path comparison is used to preserve structural diversity among attack paths.

Experiments were conducted using ADSynth-based Active Directory attack graph datasets, including r500, r1000, and r2000 environments. Experimental results showed that the proposed method discovered significantly more diverse attack paths than reinforcement learning–based approaches such as SARSA and ShotFlex-style methods. In particular, the proposed framework discovered 14 distinct attack paths in the r2000 environment, while RL-based methods often converged to only a single route. Furthermore, the Two-Stage Local Search process improved attack success probability while simultaneously reducing attack cost and detectability. These results demonstrate that the proposed framework can effectively improve both diversity and efficiency in large-scale Active Directory attack graph analysis.


2. Summary of the Data Generation Process

The data generation process is one of the core components of the paper’s experimental framework. To construct realistic Active Directory attack graph environments, the study uses an ADSynth-based synthetic data generation approach. ADSynth is a framework designed to automatically generate large-scale synthetic Active Directory attack graphs that resemble real enterprise environments. It can generate realistic distributions of user accounts, computer systems, group structures, privilege delegation relationships, and security policies. The paper utilizes ADSynth to create attack graphs that closely mimic the hierarchical structure and permission relationships found in real-world enterprise networks.

The generated dataset is represented as a directed graph. The nodes consist of Users, Groups, Computers, Organizational Units (OUs), and Group Policy Objects (GPOs), each representing a security entity with a different role and privilege level. The edges represent privilege delegation relationships, group memberships, ACL permissions, session relationships, and lateral movement opportunities. As a result, the generated graph is not merely a random connectivity graph but a realistic attack graph capable of modeling privilege escalation and lateral movement scenarios in Active Directory environments.

Each edge additionally contains attack-related attributes. Specifically, edges include attack success probability, attack cost, and detectability values. For example, certain privilege escalation actions may have high success probabilities but also high detectability, while stealth-oriented actions may require higher cost but produce lower detection risk. These attributes are later used in the multi-objective fitness evaluation process during attack path optimization.

The experiments used three datasets with different scales. The r500 dataset contains approximately 500 computers and represents a medium-scale environment. The r1000 dataset represents a larger enterprise environment with approximately 1,000 computers, while the r2000 dataset represents a very large-scale enterprise environment containing approximately 2,000 computers. As the graph size increases, both the number of nodes and edges grow rapidly, causing the attack path search space to expand exponentially. The paper uses these datasets to evaluate the scalability and exploration capability of the proposed framework under increasingly complex network conditions.

An important characteristic of the data generation process is that it does not simply produce random graphs. Instead, it incorporates realistic enterprise security structures and hierarchical privilege relationships. Highly privileged groups such as Domain Admins are sparsely distributed, while ordinary user accounts are assigned restricted privileges within specific organizational structures. Group memberships, ACL relationships, and delegation structures are generated in a hierarchical manner similar to real Active Directory environments. This design allows the generated attack graphs to support realistic privilege escalation and lateral movement analysis for cybersecurity experiments.
